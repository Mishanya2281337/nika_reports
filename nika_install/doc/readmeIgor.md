# Создание базы знаний для проекта **NIKA**
## Введение
Данный документ содержит мануал по созданию базы знаний для проекта **NIKA**. Здесь рассмотрено добавление базы месяцев для ответа на вопрос о сроке сдачи первой лабораторной работы.
## Поиск нужной директории
Рекомендую просмотреть мануал **"Изменение функций проекта **NIKA 0.2.2**"** для поиска нужного вам файла.

## Создание файла с базой знаний (месяцами)
Для начала создадим файл в формате **.scs** в директории:
``` 
knowledge-base/section_subject_domain_of_intelligent_study_systems/concepts/instances
```
## Основные идентификаторы семантических отношений и ролей для базы знаний OSTIS/NIKA

### Базовые сущности
- **`concept`** — обозначает *понятие* или класс объектов.  
- **`instance`** — конкретный экземпляр понятия.  

---

### Идентификаторы
- **`nrel_main_idtf`** — главный идентификатор (*основное имя объекта*).  
  Используется для связи узла с его текстовым названием.  
- **`nrel_system_idtf`** — системный идентификатор (*служебное имя внутри базы*).  
- **`nrel_idtf`** — дополнительные идентификаторы (*синонимы, альтернативные названия*).  

---

### Отношения
- **`nrel_inclusion`** — отношение включения (*элемент принадлежит множеству*).  
- **`nrel_subdividing`** — отношение деления множества на подмножества.  
- **`nrel_equivalence`** — отношение эквивалентности.  
- **`nrel_generalization`** — отношение обобщения (*наследование*).  
- **`nrel_specialization`** — отношение специализации (*частный случай*).  

---

### Атрибуты и свойства
- **`nrel_value`** — значение свойства.  
- **`nrel_measurement`** — измерение (*например, вес, длина*).  
- **`nrel_time`** — временные характеристики.  

---

### Служебные конструкции
- **`rrel_key_sc_element`** — ключевой элемент структуры.  
- **`rrel_example`** — пример для понятия.  
- **`rrel_answer`** — используется в диалоговых системах для ответа.  

Используя приведённые идентификаторы, напишем узел понятия "Февраль":
```
concept_february
=> nrel_main_idtf:
[февраль] (* <- lang_ru;; *);
[февраля]  (* <- lang_ru;; *);;

```
### Объяснение
1) Cоздаём узел понятия "Февраль" (`concept`). 

 
2) Через семантическое отношение nrel_main_idtf связываем узел concept_february с текстовыми ссылками.
3) Создаём текстовые ссылки (SC‑links) с содержимым:
```
[февраль] (* <- lang_ru;; *);
```
Запись в квадратных скобках — это содержимое ссылки.
Каждая такая запись — отдельная текстовая единица, хранящая строку.

4) Помечаем язык текста.
Конструкция (* <- lang_ru;; *) помечает соответствующую текстовую ссылку ролью/меткой lang_ru, то есть указывает, что содержимое — на русском языке. 

Делаем так с оставшимеся месяцами и сохраняем файл.
После чего пересобираем **knowledge-base** и запускаем **NIKA**.
